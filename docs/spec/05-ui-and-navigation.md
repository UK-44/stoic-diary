# 05. UI / ナビゲーション（stoic. 参考デザイン）

別アプリ「stoic.」を参考に、ダーク・余白多め・ミニマルなトーンで再構成する。
既存のダーク UI の方向性は踏襲する。

## 1. ナビゲーション（4 メニュー・レスポンシブ）

| メニュー | パス | 役割 |
| --- | --- | --- |
| **Home** | `/` | 当日（選択日）の日記を入力・編集。日付の切り替え。 |
| **Review** | `/review` | 1 週間 / 1 ヶ月の日記をまとめて見て、長スパンの振り返り。 |
| **Search** | `/search` | 過去の日記をキーワード or コンポーネントで絞り込み検索。 |
| **Settings** | `/settings` | 日記・振り返りのスタイル/コンポーネント構成、アカウント設定（目標等）、ログアウト。 |

- **レスポンシブ**: スマホ = 画面下の**フッタータブバー**、PC = **左サイドメニュー**。
- アクティブ表示は現在パスで判定。

## 2. 重要な制約：コンポーネントマスタは画面から変更不可

- コンポーネントの**マスタ定義（種類・名前・config）は UI から作成/編集しない**。
  定義は seed / ログイン時ブートストラップで固定供給する。
- ユーザーが Settings で調整できるのは:
  - **どのコンポーネントを使うか**（フォーム構成版の選択 = 時期ごと）
  - **並び順**（コンポーネントの order を並び替え）
  - 固定メッセージ（FIXED_MESSAGE）の**文面**
  - アカウント設定（目標 等）
- → 既存の `/admin/components`（マスタ CRUD）は**廃止**。構成系は `/settings` に集約。

## 3. Home 画面の構成

参考画像（stoic.）に倣う。

1. **上部: 週の日付ストリップ**
   - 1 週間（日〜土）の日付を横並び表示。前週/翌週へ移動できる。
   - **記入済みの日が一目で分かる**（チェ/ドット等のインジケータ）。今日・選択日を強調。
   - 日付をタップ → その日の日記に切り替え（`/?d=YYYY-MM-DD`）。
2. **本体: 選択日の日記**
   - **未記入**: 「**日記を書く**」ボタンを表示。押すと記入を開始。
   - **記入済み**: 記入済みの内容をそのまま表示・編集。
   - **インライン編集（Notion 風）**: 明確な編集モードは設けない。
     文字を入れたい箇所をタップするとキャレットが出て入力できる。
   - **自動保存**: 明示的な保存ボタンは置かず、変更を自動保存（デバウンス）。
     保存状態（保存中 / 保存済み）をさりげなく表示。

## 4. 未確定（追って詰める）
